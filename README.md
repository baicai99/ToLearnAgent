# LangChain & LangGraph Workflow 入门课程

本仓库是一套**面向 Agent 初学者**的 LangChain + LangGraph 入门课程，内容全部来自作者在学习与实践过程中的**真实学习笔记与实验代码**，并**免费开放给所有人共同学习**，旨在帮助你**按课程顺序完成基础入门**，少走弯路，避免在概念与工程细节上反复绕圈。

> 重要说明：本课程当前阶段**专注于 Workflow 设计与执行**，**不涉及 LLM 的自主决策能力**。后续章节将逐步引入具备决策能力的 Agent（Agent → Tool → Agent）。

## 课程定位

* 面向对象：

  * 想系统学习 LangChain / LangGraph 的开发者
  * 想从「可控流程」理解 Agent 的工程实现方式的学习者

* 学习目标：

  * 理解 LangChain 中 Tool、Message、Runnable 等核心概念
  * 理解 LangGraph 中的 State、Node、Edge、Graph 执行模型
  * 能够独立构建 **确定性 Workflow（非自主决策）**
  * 为后续 Agent 化、多工具推理打下工程基础

---

## 课程结构

课程以 **Workflow 为主线**，每一节课都是一个可运行的最小示例：

* 每个示例关注一个清晰的知识点
* 不做“兜底式封装”，鼓励暴露真实错误
* 强调状态流转与执行路径的可解释性

大致学习路径：

1. LangChain 基础组件

   * Message / Tool / Runnable
   * Tool 调用与结果回灌

2. Workflow 思维建立

   * 顺序执行
   * 条件分支
   * 明确的输入 / 输出状态

3. LangGraph 基础

   * State 定义
   * Node 与 Edge
   * Graph 编译与执行

4. 组合式 Workflow

   * 多节点协作
   * 状态在节点间的传递

> 当前阶段：**全部为确定性流程（Workflow）**，不包含模型自主判断或多轮规划。

---

## 为什么先学 Workflow，而不是 Agent？

在工程实践中：

* Agent ≠ 黑盒推理
* Agent = **Workflow + 决策能力**

如果直接从“全自动 Agent”入手，往往会：

* 无法定位错误来源
* 不理解状态是如何被修改的
* 难以控制执行路径

因此，本课程**刻意拆解 Agent 能力**：

> 先掌握 **可控、可解释、可调试的 Workflow**，再引入 LLM 决策能力。
